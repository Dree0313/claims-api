## Overview

This project simulates a **real-world insurance claims processing system** using modern Java technologies while mimicking legacy mainframe COBOL workflows.  
It demonstrates a full workflow:

1. Java REST API receives insurance claims.
2. Claims are processed through a **COBOL simulator** (implemented in Java for Windows compatibility).
3. Fixed-width files simulate mainframe output.
4. Claims are stored in an **H2 in-memory database** (simulating DB2 functionality).
5. API endpoints allow retrieving processed claims and outputs.

This project mirrors how enterprise insurance systems handle mission-critical applications using both **modern and legacy technologies**.

---

## Features

- **Java Spring Boot REST API**
  - `POST /claims/process` → Submit claim for processing
  - `GET /claims/output` → Retrieve COBOL-like output file
  - `GET /claims/history` → Retrieve saved claims from H2 database
- **COBOL Simulation (Java)**
  - Reads claims from text file
  - Applies business rules for payout calculations
  - Writes fixed-width output file
- **Database Simulation**
  - H2 in-memory database for claims persistence
  - JPA/Hibernate used for repository pattern
- **File-Based Workflow**
  - Simulates sending data from Java → COBOL → Java
  - Fixed-width files mimic legacy mainframe output

## Getting Started

### Prerequisites

- Java 17+
- Maven
- IntelliJ IDEA or any Java IDE
- Git (optional, for cloning repository)

### Installation

1. Clone the repository:

```bash
git clone https://github.com/Dree0313/claims-api.git
cd claims-api
````

2. Build and run the application:

```bash
mvn clean install
mvn spring-boot:run
```

3. Access API:

* REST API: `http://localhost:8080`
* H2 Console: `http://localhost:8080/h2-console`
  JDBC URL: `jdbc:h2:mem:claimsdb`, User: `SA`, Password: *(empty)*

---

## Usage

1. **Process Claims**

   ```bash
   POST http://localhost:8080/claims/process
   Body (JSON):
   {
       "name": "John Doe",
       "policyNumber": "POL123456",
       "claimAmount": 5000,
       "claimType": "ACCIDENT"
   }
   ```

2. **Get COBOL Output**

   ```bash
   GET http://localhost:8080/claims/output
   ```

   * Returns fixed-width file content, simulating mainframe output.

3. **Get All Claims**

   ```bash
   GET http://localhost:8080/claims/history
   ```
